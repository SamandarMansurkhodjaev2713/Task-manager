# Telegram Task Bot

## Развернутое описание функционала

Этот документ описывает функциональные возможности бота в текущем виде простым и понятным языком.  
Он подготовлен для презентации продукта заказчику, руководителю или потенциальному покупателю.

---

## 1. Что это за бот

Telegram Task Bot — это бот для постановки, распределения и сопровождения задач внутри команды прямо в Telegram.

Он помогает:
- быстро ставить задачи коллегам;
- назначать исполнителей;
- контролировать сроки;
- видеть статус работы без лишних уточнений;
- вести коммуникацию по задаче в одном месте;
- не терять задачи, комментарии и договорённости;
- давать менеджеру понятный обзор по команде.

Бот ориентирован на ежедневную рабочую коммуникацию и делает управление задачами проще без перехода в сложные внешние системы.

---

## 2. Для кого подходит

Бот особенно полезен для:
- небольших и средних команд;
- руководителей и менеджеров проектов;
- отделов дизайна, разработки, маркетинга, продаж, операционных команд;
- команд, которые уже активно работают в Telegram и хотят управлять задачами прямо в привычной среде.

---

## 3. Основная ценность продукта

Бот решает сразу несколько задач:
- превращает обычную переписку в понятные и зафиксированные задачи;
- показывает, что именно требует внимания прямо сейчас;
- помогает руководителю не терять контроль над загрузкой команды;
- делает постановку задач удобной даже для людей, которые не любят сложные таск-трекеры;
- снижает риск “я не видел”, “я забыл”, “не понял, что от меня нужно”, “кому это вообще назначено”.

---

## 4. Работа с ролями

В боте предусмотрены разные уровни работы в зависимости от роли пользователя.

### Обычный участник команды

Может:
- создавать задачи;
- получать задачи в работу;
- видеть свои задачи;
- видеть задачи, которые он поставил другим;
- менять статус своих задач;
- оставлять комментарии;
- сообщать о блокерах;
- отправлять задачу на проверку;
- возвращать задачу в работу при необходимости.

### Менеджер

Дополнительно получает:
- доступ к командным спискам задач;
- отдельный экран внимания менеджера;
- обзор задач команды;
- контроль задач на проверке;
- видимость блокеров;
- видимость проблем с доставкой задач исполнителям;
- обзор задач с риском по срокам;
- возможность переназначать задачи.

### Администратор

Дополнительно может выполнять служебные действия, связанные с поддержкой и актуализацией данных сотрудников.

---

## 5. Главное меню

После запуска бот показывает понятное стартовое меню.

Для обычного пользователя доступны основные разделы:
- **Мой фокус**
- **Создать задачу**
- **Мои задачи**
- **Созданные мной**
- **Моя статистика**
- **Профиль**

Для менеджера дополнительно доступны:
- **Командные задачи**
- **Inbox менеджера**
- **Командная статистика**

Это меню построено так, чтобы человек сразу понимал, куда нажимать в зависимости от своей цели: поставить задачу, посмотреть свою загрузку, отследить делегированные задачи или проверить ситуацию по команде.

---

## 6. Создание задач

В боте предусмотрено два основных режима создания задачи.

### 6.1. Быстрое создание

Подходит для ситуации, когда задача уже сформулирована и её нужно просто быстро поставить.

Пользователь может:
- отправить обычное текстовое сообщение;
- использовать голосовое сообщение как источник постановки задачи.

Это удобно, когда нужно быстро зафиксировать поручение без лишних шагов.

### 6.2. Пошаговый мастер

Подходит для ситуаций, когда задачу хочется оформить аккуратно и ничего не забыть.

Бот по шагам помогает:
1. указать исполнителя;
2. ввести описание задачи;
3. задать срок или выбрать вариант без срока;
4. проверить карточку перед созданием;
5. подтвердить постановку.

Такой сценарий особенно полезен для менеджеров и для задач, где важны точность формулировки и дедлайн.

---

## 7. Назначение исполнителя

Бот умеет назначать задачу несколькими способами:
- по `@username`;
- по данным сотрудника;
- по имени или другому узнаваемому идентификатору;
- через уточнение, если совпадений несколько.

Если бот понимает исполнителя не однозначно, он не делает рискованное назначение “наугад”, а предлагает уточнить выбор.

Если задача назначается самому себе, это также поддерживается как отдельный рабочий сценарий.

---

## 8. Что происходит, если исполнитель ещё не запускал бота

Это важный рабочий сценарий.

Если исполнитель найден, но ещё не начал диалог с ботом:
- задача всё равно создаётся;
- автор видит, что задача зафиксирована;
- бот показывает, что уведомление пока ожидает активации со стороны исполнителя;
- как только пользователь начнёт взаимодействие с ботом, доставка может быть выполнена корректно.

Таким образом задача не теряется, а её состояние остаётся прозрачным.

---

## 9. Система идентификации задач

Каждая задача имеет короткий понятный код, например:
- `T-0001`
- `T-0142`
- `T-1543`

Этот код используется в пользовательском интерфейсе:
- в списках;
- в карточке задачи;
- в сообщениях;
- при переходе к задаче;
- в командах и поиске.

Это делает работу с задачами намного понятнее и удобнее, чем длинные технические идентификаторы.

---

## 10. Карточка задачи

Карточка задачи — это основной экран работы с конкретной задачей.

### 10.1. Компактная карточка

По умолчанию бот показывает краткую и удобную карточку, в которой видны:
- название задачи;
- короткий код задачи;
- текущий статус;
- срок;
- исполнитель;
- состояние доставки уведомления;
- главное следующее действие;
- краткое превью содержания.

Такой формат помогает быстро понять суть задачи без перегруза информацией.

### 10.2. Расширенная карточка

При необходимости можно открыть подробный режим, где дополнительно доступны:
- полное описание;
- ожидаемый результат;
- критерии приёмки;
- последние комментарии;
- история изменений.

То есть по умолчанию карточка остаётся лёгкой, а детали открываются только тогда, когда они действительно нужны.

---

## 11. Статусы задач

Бот поддерживает жизненный цикл задачи от постановки до завершения.

Пользователь видит статусы в человеко-понятном виде, без технических обозначений.

В рабочем процессе бот поддерживает состояния:
- новая задача;
- задача отправлена исполнителю;
- в работе;
- заблокирована;
- на проверке;
- завершена;
- отменена.

Переходы между статусами контролируются логично и последовательно, чтобы снизить риск случайных или бессмысленных действий.

---

## 12. Главное действие по задаче

Бот не просто показывает список кнопок, а подсказывает, какое действие сейчас главное.

Например:
- если задача новая и назначена пользователю — главным действием будет взять её в работу;
- если задача уже выполняется — следующим действием может быть отправка на проверку;
- если задача находится на проверке — менеджеру или автору будет удобно быстро принять результат;
- если задача заблокирована — бот делает акцент на действиях по разблокировке.

Это повышает скорость работы и уменьшает количество неверных действий.

---

## 13. Подтверждение опасных действий

Для действий, которые могут повлиять на результат работы, бот запрашивает отдельное подтверждение.

Это касается, например:
- отмены задачи.

Такой подход снижает риск случайных отмен и ошибочных нажатий.

---

## 14. Работа с комментариями

Внутри задачи можно вести рабочую коммуникацию.

Комментарии нужны для:
- уточнений;
- передачи промежуточной информации;
- фиксации договорённостей;
- коротких апдейтов по задаче.

Это помогает не терять важный контекст и держать обсуждение в привязке к конкретной задаче.

---

## 15. Работа с блокерами

Если задача упёрлась в проблему, исполнитель может отметить её как заблокированную.

Это позволяет:
- сразу показать, что задача не просто “стоит”, а реально упёрлась в препятствие;
- сделать блокер видимым для автора и менеджера;
- ускорить реакцию на проблемы;
- не смешивать обычную задержку и настоящий стоп-фактор.

Для менеджера такие задачи попадают в отдельную зону внимания.

---

## 16. Проверка результата и review flow

Когда задача готова, её можно перевести в состояние проверки.

Это даёт возможность:
- отделить “я закончил” от “работа официально принята”;
- организовать понятный цикл согласования результата;
- упростить работу менеджера или автора задачи;
- фиксировать момент завершения более осознанно.

При необходимости задача может быть возвращена обратно в работу.

---

## 17. Переназначение задачи

Если меняется исполнитель, задачу можно переназначить.

Это полезно, когда:
- меняется загрузка команды;
- задача изначально ушла не тому человеку;
- нужно быстро передать работу другому сотруднику;
- задача переходит между ролями внутри команды.

---

## 18. История изменений

По каждой задаче хранится история ключевых действий.

Это позволяет видеть:
- когда задача была создана;
- когда менялся статус;
- когда она была отменена;
- какие важные изменения происходили по ходу работы.

История помогает восстанавливать ход событий и делает работу прозрачнее для автора, исполнителя и менеджера.

---

## 19. Списки задач

Бот поддерживает несколько рабочих списков, каждый из которых решает свою задачу.

### 19.1. Мой фокус

Это основной ежедневный экран для исполнителя.

Он помогает быстро увидеть:
- что просрочено;
- что требует внимания сегодня;
- что ждёт действий пользователя;
- что заблокировано;
- что находится на проверке;
- какие задачи сейчас в активной работе.

Это экран для приоритизации дня.

### 19.2. Мои задачи

Полный список задач, назначенных пользователю.

Подходит для просмотра личного рабочего пула целиком.

### 19.3. Созданные мной

Раздел для контроля задач, которые пользователь поставил другим людям или себе.

Подходит для делегирования и отслеживания результата.

### 19.4. Командные задачи

Расширенный командный список для менеджера.

Позволяет смотреть на задачи уже не только как исполнитель, а как руководитель, которому важно видеть общую картину.

### 19.5. Inbox менеджера

Это отдельный экран внимания для менеджера, где собраны самые важные сигналы:
- задачи, которые ждут решения или проверки;
- задачи с блокерами;
- задачи, которые ещё не доставлены исполнителю;
- задачи с риском по сроку;
- задачи, в которых давно не было движения.

Это делает управление командой более проактивным.

---

## 20. Уведомления и прозрачность доставки

В боте отдельно отслеживается не только состояние задачи, но и состояние доставки уведомления.

Пользователь может видеть, было ли уведомление:
- доставлено;
- поставлено в очередь;
- отложено до старта исполнителя;
- отправлено на повтор;
- не доставлено.

Это очень важное преимущество: бот не создаёт ложного ощущения, что исполнитель уже всё увидел, если по факту это ещё не произошло.

---

## 21. Честная работа с дублями

Если одно и то же сообщение уже было обработано и задача по нему уже существует, бот не создаёт дубликат незаметно.

Вместо этого он:
- честно сообщает, что такая задача уже была создана ранее;
- открывает существующую карточку;
- не вводит пользователя в заблуждение сообщением “успешно создано”, если новая задача фактически не появилась.

Это повышает доверие к системе и снижает путаницу.

---

## 22. Удобство текстов и интерфейса

Отдельное внимание уделено удобству ежедневного использования.

Интерфейс ориентирован на:
- короткие понятные сообщения;
- живой и дружелюбный тон;
- минимизацию канцелярита;
- понятные кнопки;
- логичную группировку действий;
- быстрый возврат назад;
- снижение информационного шума;
- ясную формулировку того, что происходит прямо сейчас.

То есть бот не выглядит как технический инструмент для специалистов, а воспринимается как понятный рабочий помощник.

---

## 23. Напоминания и регулярные сводки

Бот поддерживает фоновые сценарии сопровождения задач.

Среди них:
- напоминания по дедлайнам;
- оповещения о просроченных задачах;
- ежедневные сводки;
- уведомления о важных изменениях по задачам.

Это помогает команде не терять сроки и держать в поле зрения важные задачи.

---

## 24. Статистика

В боте доступны экраны со статистикой.

Они позволяют смотреть:
- личную статистику пользователя;
- командную статистику для менеджера.

Это полезно для оценки загрузки, активности и общей картины по работе команды.

---

## 25. Поиск и переход к задаче

Благодаря коротким кодам задач переход к нужной задаче становится намного удобнее.

Пользователю проще:
- найти нужную задачу;
- сослаться на неё в переписке;
- открыть её по коду;
- быстро сверить статус без необходимости работать с длинными идентификаторами.

---

## 26. Сценарии ежедневного использования

### Для сотрудника

Типичный сценарий выглядит так:
1. открыть **Мой фокус**;
2. увидеть, что требует внимания сегодня;
3. открыть нужную задачу;
4. взять её в работу или обновить статус;
5. при необходимости оставить комментарий или отметить блокер;
6. отправить задачу на проверку после завершения.

### Для менеджера

Типичный сценарий выглядит так:
1. открыть **Inbox менеджера**;
2. увидеть задачи с рисками и блокерами;
3. открыть задачу, которая требует решения;
4. проверить статус и детали;
5. принять результат, вернуть в работу или переназначить исполнителя;
6. при необходимости пройтись по командным задачам и статистике.

---

## 27. Что особенно важно с точки зрения бизнеса

С практической точки зрения бот даёт бизнесу следующие преимущества:
- прозрачность постановки задач;
- снижение потерь в коммуникации;
- более дисциплинированное управление сроками;
- удобную делегацию;
- контроль по исполнителям;
- отдельную управленческую точку обзора;
- меньше хаоса в рабочих чатах;
- фиксацию контекста по задаче в одном месте;
- понятный и быстрый вход для новых сотрудников.

---

## 28. Итог

Telegram Task Bot — это рабочий инструмент для командного управления задачами прямо внутри Telegram.

В текущем виде он уже покрывает полный базовый цикл работы с задачами:
- постановка;
- назначение;
- доставка;
- сопровождение;
- комментарии;
- блокеры;
- проверка;
- завершение;
- контроль со стороны менеджера;
- напоминания и регулярные обзоры.

Главное преимущество бота в том, что он сочетает:
- привычный интерфейс Telegram;
- понятную ежедневную работу для сотрудника;
- управленческую прозрачность для руководителя;
- аккуратную и удобную логику взаимодействия без лишней перегрузки.

Если нужно, на основе этого документа можно отдельно подготовить ещё и:
- короткую продающую версию на 1 страницу;
- коммерческую презентацию по блокам;
- PDF-версию в более “презентационном” стиле;
- версию для лендинга или коммерческого предложения.
